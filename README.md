# hit-movies-tracker

Генератор HTML-страницы топ-100 HD-фильмов с Pirate Bay, обогащённой рейтингом IMDB, жанрами, постерами и ссылками на YouTube-трейлеры. Без API-ключей.

## Возможности

- Скачивает актуальный список топ-100 HD-Movies с Pirate Bay
- Находит каждый фильм в IMDB: рейтинг, жанр, постер, актёров
- Ищет трейлер на YouTube и даёт прямую ссылку на видео
- Два режима: **список** (таблица) и **плитка** (карточки)
- Мобильный режим **📱/🖥** — принудительные мобильные стили для предпросмотра
- Сортировка кликом на заголовки: **Название**, **Размер**, **Дата**, **Рейтинг** — синхронизирована между списком и плиткой
- Фильтр по жанрам — динамический `<select>`, собирается из видимых фильмов
- Автоскрытие ужасов и секс-контента
- ✕ для скрытия фильма и всех одноимённых (без подтверждения)
- На постере — полупрозрачная кнопка ▶ для трейлера
- Раскрывающиеся детали: постер, жанр, рейтинг, актёры
- Кеширование всего — повторная генерация за 0.2 сек
- Инкрементальный кеш IMDB: только ID из TPB (~50-200KB вместо 1GB)
- Хеш страницы TPB: если список фильмов не изменился — пропускает обновление
- Синхронизация: новые добавляются, удалённые — убираются вместе с постерами

```powershell

cd hit-movies-tracker
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Использование

**Однократная генерация (из кеша):**

```powershell
python generate_page.py
```

**Обновление с Pirate Bay (с кешем):**

```powershell
python generate_page.py --refresh
# → скачивает свежий список
# → если фильмы те же — использует кеш без перегенерации
# → иначе добавляет новые, убирает пропавшие, обогащает IMDB
```

**Полная перегенерация без кеша:**

```powershell
python generate_page.py --refresh --nocache
# → принудительно обновляет всё, даже если список не изменился
```

**Через HTTP-сервер (с кнопкой 🔄 на странице):**

```powershell
python server.py
# → http://localhost:8765
```

**Быстрый запуск:**

```powershell
refresh.bat
# → запускает generate_page.py --refresh и открывает torrents.html
```

## Как это работает

### Без API-ключей

1. **Pirate Bay** — парсится HTML страницы `/top/207`
2. **IMDB ID** — через autocomplete API `v2.sg.media-imdb.com` (поиск по названию с предпочтением результата с постером)
3. **Рейтинг и жанр** — из официальных датасетов IMDB (`title.ratings.tsv.gz`, `title.basics.tsv.gz`, ~8MB)
4. **Постер** — скачивается с IMDB, сохраняется локально в `posters/`
5. **Трейлер** — поиск через YouTube, ссылка на прямое видео

### Оптимизация `--refresh`

При `--refresh` сперва вычисляется SHA256-хеш таблицы фильмов со страницы TPB. Если хеш совпадает с сохранённым (`piratebay_hash.txt`) — вся обработка (IMDB датасеты, поиск, обогащение) пропускается, используется кеш. Принудительно обойти — `--nocache`.

### Кеши

| Файл | Назначение |
|------|-----------|
| `imdb_ratings_cache.json` | Рейтинги IMDB (только нужные ID, ~50-200KB) |
| `imdb_basics_cache.json` | Жанры IMDB (только нужные ID, ~50-200KB) |
| `imdb_search_cache.json` | Результаты поиска название→ID + постер + актёры |
| `youtube_cache.json` | Ссылки на YouTube-трейлеры |
| `piratebay_page.html` | Слепок страницы Pirate Bay |
| `piratebay_hash.txt` | SHA256-хеш таблицы фильмов для оптимизации `--refresh` |
| `torrents_data.json` | Все обогащённые данные по торрентам |
| `posters/` | Локальные копии постеров |
